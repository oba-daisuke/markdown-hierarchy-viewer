## 参加すべき定例ミーティング

| ミーティング | 頻度 | 目的 |
|------------|------|------|
| 全体定例 | 隔週木曜 | 会社全体の状況把握 |
| チーム朝会 | 毎日 | 当日タスク共有 |
| スプリントプランニング | 2週間ごと | 次スプリントの計画 |
| レトロスペクティブ | 2週間ごと | 振り返り・改善 |

## 重要なSlackチャンネル

| チャンネル | 用途 |
|-----------|------|
| `#general` | 全社アナウンス |
| `#dev` | 開発全般の議論 |
| `#incident` | 障害対応 |
| `#releases` | リリース通知 |
| `#random` | 雑談 |

## 困ったときは

技術的な質問 → `#dev` へ（恥ずかしくないです！）  
プロセスの質問 → メンターか上長へ

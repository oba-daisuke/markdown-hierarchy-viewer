# インシデント対応手順

## 重大度レベル

| レベル | 定義 | 初動目標 | 対応チャンネル |
|-------|------|---------|--------------|
| P0 | 全ユーザー影響・売上停止 | 5分以内 | #incident-p0 |
| P1 | 主要機能障害・一部ユーザー影響 | 15分以内 | #incident |
| P2 | 機能縮退・代替手段あり | 1時間以内 | #dev-alerts |
| P3 | 軽微な不具合・ユーザー影響小 | 翌営業日 | Jira ticket |

## P0/P1 対応フロー

```
1. 検知 (監視アラート or ユーザー報告)
      ↓
2. 初動確認 (5分以内)
   - 影響範囲の特定
   - 重大度の判定
   - Slackチャンネル作成 + 関係者招集
      ↓
3. コミュニケーション
   - ステータスページ更新
   - 社内Slackで状況共有
   - (必要に応じて) カスタマーへの通知
      ↓
4. 調査・対応
   - インシデントコマンダーを1名指名
   - 原因調査と暫定対応を並行実施
      ↓
5. 解消確認
   - メトリクスが正常値に戻ったことを確認
   - ステータスページ: 復旧済みに更新
      ↓
6. 事後レビュー (48時間以内)
   - ポストモーテム作成
   - 再発防止策の策定
```
